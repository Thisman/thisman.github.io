# AGENTS.md - Swap-Tac-Toe

## Описание приложения

Приложение «Swap-Tac-Toe» — расширенная версия крестиков-ноликов (Connect-N) с дополнительной механикой swap. Игроки X и O по очереди либо ставят метку в пустую клетку (Place), либо меняют местами две соседние клетки с разными метками (Swap). Цель — собрать линию из N меток. Поддерживаются режимы PvP и PvC (против компьютера).

## Основные механики

### 1. Типы ходов
- **Place**: поставить метку X или O в любую пустую клетку
- **Swap**: выбрать занятую клетку, затем кликнуть на соседнюю (включая диагональ) с другой меткой — они меняются местами

### 2. Правила победы
- Победа: собрать непрерывную линию длиной N (горизонталь, вертикаль, диагональ)
- Проверка: сначала проверяется победа сделавшего ход игрока, затем — не создал ли swap победу сопернику
- Ничья: вручную, при исчерпании swap-лимита после заполнения доски, или по правилу трёхкратного повтора

### 3. Настройки игры
- Размер поля: 3–15 строк × 3–15 столбцов
- Длина линии (N): 3–max(rows, cols)
- Лимит swap после заполнения доски (на игрока)
- Пресеты: 3×3 N=3, 5×5 N=4, 10×10 N=5, 15×15 N=5

### 4. Режим PvC (против компьютера)
- Человек играет за X, компьютер за O
- ИИ реализован в `bot.js`
- Компьютер делает ход с задержкой (`computerBusy` флаг предотвращает двойные ходы)

### 5. Многоэкранный интерфейс
- Экран 1 — настройки (`#screenSettings`)
- Экран 2 — игра (`#screenGame`) с полем и статусбаром
- Экран 3 — результаты (`#screenResults`) со статистикой

## Ключевые функции

### `startGame()` (game.js)
Инициализирует доску, рендерит DOM-клетки, запускает игру с текущими настройками.

### `handleCellClick(r, c)` (game.js)
Обрабатывает клик по клетке: определяет тип хода (Place или первый шаг Swap), применяет ход.

### `checkWin(board, player, R, C, K)` (game.js)
Проверяет победу игрока: сканирует все 4 направления по всему полю.

### `applySwap(r1, c1, r2, c2)` (game.js)
Меняет местами содержимое двух клеток, обновляет DOM и проверяет обе победы.

### `showResults(winner, reason)` (game.js)
Переключает на экран результатов, заполняет статистику.

### Бот (bot.js)
Вычисляет оптимальный ход для O с учётом Place и Swap механик.

## Правила работы

### 1. Состояние игры
- `board[r][c]` — строки `""`, `"X"`, `"O"`
- `swapFirst` — первая клетка при Swap (`{r, c}` или `null`)
- `blockedSwapCells` — запрет повторного свапа той же пары на следующем ходу
- `gameMode` — `"pvp"` или `"pvc"`

### 2. HTML-структура
- Три `<section>` с классом `screen`, активный получает класс `active`
- Поле рендерится динамически: DOM-элементы клеток в `#board`
- Скрытые `<input type="hidden">` хранят значения настроек

### 3. CSS и стилизация
- Общие стили из `/shared/common.css`
- CSS-переменные для цветов и размеров
- Класс `selected` на выбранной клетке при Swap, `win-cell` на победной линии

### 4. Навигация
- Ссылка на главную страницу проекта
- Ссылка на GitHub репозиторий в `<footer>`

## Зависимости

- **Общие стили**: `/shared/common.css`
