# AGENTS.md - My Valentine (Арканоид)

## Описание приложения

Приложение «Будь моей валентинкой» — арканоид-игра, приуроченная ко Дню святого Валентина. Игрок управляет платформой и разбивает блоки, выложенные в форме сердца. По мере уничтожения блоков проявляется валентинское послание, а при победе открывается изображение-валентинка.

## Основные механики

### 1. Игровое поле
- Canvas 900×760 пикселей
- Блоки расположены по паттерну `HEART_PATTERN` — двумерный булевый массив 8×11
- Позиции блоков вычисляются динамически при `initBricks()` с центрированием по ширине

### 2. Физика мяча
- Обновление на основе `deltaTime` через `requestAnimationFrame`
- Скорость мяча фиксирована (`ballSpeed: 420`), нормализуется при отражении от платформы
- Угол отражения от платформы зависит от точки удара (`hitPoint`): `bounceAngle = hitPoint * maxBounceAngle`
- Столкновение с блоком: `circleRectCollision` + `bounceFromBrick` (overlap-based reflection)
- `deltaTime` ограничен до 32 мс для предотвращения туннелирования при потере фокуса

### 3. Управление
- Клавиши `←/→`, `A/D`, `Ф/В` (русская раскладка) — движение платформы
- `Space` — запуск мяча или перезапуск после окончания игры
- Кнопка «Перезапуск» — сброс всего состояния

### 4. Прогресс победы
- Сообщение «Будь моей валентинкой» появляется постепенно по мере уничтожения блоков
- Прозрачность = `1 - remainingBricks / totalBricks`
- При победе: CSS-класс `is-active` на `winImageEl` показывает изображение-сердце

### 5. Жизни и состояния игры
- 3 жизни (`lives: 3`), отображаются в HUD
- Состояния: `"playing"`, `"won"`, `"lost"`
- Потеря жизни: мяч упал ниже `GAME_HEIGHT`

## Ключевые функции

### `initBricks()` (index.js)
Создаёт массив блоков по `HEART_PATTERN`, вычисляя координаты с центрированием.

### `circleRectCollision(circle, rect)` (index.js)
Проверяет столкновение шара с прямоугольником через nearest-point алгоритм.

### `bounceFromBrick(brick)` (index.js)
Инвертирует `vx` или `vy` в зависимости от overlap по X и Y.

### `updateBall(dt)` (index.js)
Обновляет позицию мяча, обрабатывает все коллизии: стены, блоки, платформа.

### `updateMessageOpacity()` (index.js)
Обновляет прозрачность надписи в зависимости от прогресса уничтожения блоков.

### `loop(timestamp)` (index.js)
Главный игровой цикл: вычисляет `dt`, вызывает `update` и `draw`.

## Правила работы

### 1. Единый файл логики
- Вся игровая логика в `index.js`: физика, рендеринг, ввод
- Без внешних библиотек — только Canvas 2D API
- Глобальные переменные хранят состояние игры

### 2. Canvas рендеринг
- `ctx.clearRect` + последовательный рендер: блоки → платформа → мяч
- Блоки с `alive: false` пропускаются
- Canvas имеет фиксированный размер (не адаптивный)

### 3. Обработка ввода
- `keydown/keyup` записывают `keys.left/right`
- На `blur` обнуляются все клавиши для предотвращения залипания
- Поддержка русской раскладки: `ф` = left, `в` = right

### 4. Навигация
- Нет ссылок на другие приложения (standalone-игра)
- Встроенный CSS без внешних стилей

## Зависимости

- **Нет внешних зависимостей** — чистый Canvas 2D + Vanilla JS
