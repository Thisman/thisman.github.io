# AGENTS.md - The Maze (Dual Grid Puzzle)

## Описание приложения

Приложение «The Maze» (Dual Grid Puzzle) — головоломка на сетке, где два объекта (A и B) управляются одновременно одними клавишами. Задача — привести оба объекта в целевые позиции за ровно T шагов, избегая препятствий. Включает встроенный редактор уровней и инструменты для генерации новых уровней.

## Основные механики

### 1. Двойное движение
- Объекты A и B перемещаются одновременно при каждом нажатии клавиши
- Управление: стрелки или WASD
- Объект не может выйти за границу сетки (`collision: "bump"` — он остаётся на месте)

### 2. Уровни
- Формат: JSON `dualgrid-level@2`
- Параметры уровня: размер сетки (`w`, `h`), начальные позиции A и B, цели, препятствия
- `T` — точное количество шагов для победы (при `winOnExactT: true`)
- `mapsCount` — количество карт в уровне
- Уровни хранятся в `levels/level_XXXX.json`, 11 уровней в проекте

### 3. Состояния игры
- Загрузка уровня с минимальной задержкой 1000 мс (индикатор `#loading`)
- Overlay с результатом: победа → следующий уровень, поражение → перезапуск
- Счётчик шагов отображается в статусбаре

### 4. Рендеринг
- Canvas 2D рендерер (`renderer_canvas.js`)
- Отображение: сетка, объекты A и B, цели, препятствия
- Hover-подсказки при наведении мыши на клетки (tooltip)
- Иконки вращения (`ROTATION_ICON`) для визуального контекста

### 5. Редактор уровней
- Отдельная страница `editor.html` с логикой в `src/editor.js`
- Позволяет создавать и редактировать уровни визуально
- Экспорт уровней в JSON-формат

## Ключевые функции

### `createGameSession(level)` (src/game/session.js)
Создаёт игровую сессию из runtime-уровня: начальное состояние, история ходов.

### `buildRuntimeLevel(spec)` (src/core/level.js)
Преобразует JSON-спецификацию уровня во внутреннее представление.

### `validateLevelSpec(spec)` (src/core/level.js)
Проверяет корректность JSON-уровня перед использованием.

### `createRenderer(canvas)` (src/renderer_canvas.js)
Создаёт объект рендерера с методом `render(gameState)` для отрисовки сцены.

### `bindInput(callback)` (src/input.js)
Привязывает обработчики клавиатуры и возвращает функцию отвязки.

### Загрузка уровней (src/main.js)
Асинхронная загрузка JSON через `fetch`, кеширование в `Map`, переход между уровнями.

## Правила работы

### 1. Модульная архитектура
- Чистые ES6-модули (`"type": "module"` в package.json)
- Разделение: `core` (чистая логика) / `game` (сессия) / рендерер / ввод
- `core/` не зависит от DOM — только чистые функции

### 2. Формат уровней
- Все уровни в `levels/level_NNNN.json` с нумерацией из 4 цифр
- Поле `format: "dualgrid-level@2"` обязательно для валидации
- Препятствия — массив координат `[x, y]` в `obstacles`
- Позиции A и B — массивы `[x, y]`

### 3. Игровой цикл
- `requestAnimationFrame` для рендеринга
- Отдельный объект `anim` для управления анимационным циклом
- Минимальное время загрузки 1000 мс для предотвращения мигания

### 4. Тестирование и инструменты
- Тесты (`tests/core.test.mjs`) покрывают ядро логики движения
- `tools/check-level.mjs` — CLI для валидации уровней
- `tools/gen-levels.mjs` — CLI для автогенерации уровней

## Зависимости

- **Font Awesome** 6.5.2 (CDN): иконки интерфейса
- **Нет JS-зависимостей** — чистые ES6 модули
