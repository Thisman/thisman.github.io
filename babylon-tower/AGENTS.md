# AGENTS.md - Pentagonal Prism Slider

## Описание приложения

Приложение «Pentagonal Prism Slider» — пазл-головоломка в 3D на основе пятиугольной призмы. Игрок решает пространственную головоломку, перемещая чипы по граням и слоям призмы, чтобы каждая грань содержала чипы одного цвета. Аналог кубика Рубика в форме пятиугольной призмы.

## Основные механики

### 1. Структура состояния
- Сетка `state[layer][face]`: 5 слоёв × 5 граней
- Значения: `0` (EMPTY), `1` (C1), `2` (C2), `3` (C3), `4` (C4)
- Инвариант: ровно 5 пустых клеток в любой момент

### 2. Ходы
- **SLIDE**: чип скользит вверх (`dir=-1`) или вниз (`dir=+1`) по грани в пустую ячейку
- **ROTATE**: слой поворачивается по часовой (`CW`) или против часовой (`CCW`) стрелки на 72°
- Оба типа ходов записываются в историю для Undo

### 3. Победное условие
- Грань 0 полностью пустая, грани 1–4 содержат только чипы соответствующего цвета
- Функция `isSolved(state)` проверяет состояние после каждого хода

### 4. Управление
- Мышь: перетащить — вращение камеры, колёсико — зум
- Клик по чипу → клик по цели — перемещение
- Кнопки HUD: `Rotate CW` / `CCW` по слоям, `Reset`, `Scramble`, `Undo`
- `Esc` — снять выделение с чипа

### 5. Рандомизация
- `scramble(state, steps)` применяет случайные допустимые ходы
- Фильтрует обратные ходы, чтобы избежать back-and-forth

```

```

## Ключевые функции

### `createSolvedState()` (logic.js)
Создаёт эталонное решённое состояние пятиугольной призмы.

### `applyRotate(state, layer, direction)` (logic.js)
Вращает один слой на 72° в указанном направлении, модифицируя `state`.

### `applySlide(state, face, layer, dir)` (logic.js)
Перемещает чип по грани в пустую ячейку, предварительно проверив `canSlide()`.

### `scramble(state, steps)` (logic.js)
Перемешивает головоломку случайными ходами, исключая немедленные обратные.

### `isSolved(state)` (logic.js)
Проверяет, соответствует ли текущее состояние победному.

## Правила работы

### 1. Модульная архитектура
- Три ES6-модуля (`logic`, `render`, `input`) подключаются через `<script type="module">`
- Используется нативный `importmap` для разрешения зависимости `three`
- Модули общаются через экспорт/импорт, без глобальных переменных

### 2. Состояние игры
- Одиночный объект `state` — источник истины
- История ходов хранится в массиве для Undo
- `cloneState(state)` используется перед каждым изменением для сохранения истории

### 3. Рендеринг
- Все 3D-объекты управляются через Three.js
- Canvas занимает 100vw × 100vh
- HUD — фиксированные HTML-элементы поверх canvas

### 4. Обработка событий
- `mousedown/mousemove/mouseup` — вращение камеры и выбор чипа
- `wheel` — зум камеры
- `click` — выбор чипа и цели для слайда
- Кнопки HUD имеют `id`-атрибуты для привязки в `input.js`

## Зависимости

- **Three.js** r0.160.0 (CDN via unpkg, importmap): 3D-рендеринг WebGL
- **Three.js OrbitControls** (addons): управление камерой
